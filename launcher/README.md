Launcher README
--------------

Use the launcher scripts to create a double-clickable app that starts the server and opens a public tunnel (ngrok by default).

Files:
- `launcher/start-app.js` — Node launcher that spawns the server with the configured tunnel, waits for `LIVE_DEMO_URL.txt`, and opens the URL.
- `launcher/launcher-config.json` — Edit this to set `preferredTunnel: "ngrok"` and `ngrokAuthtoken` for your machine.
- `mac/run-localserver.command` — Double-click on macOS (make executable: `chmod +x mac/run-localserver.command`).
- `windows/run-localserver.bat` — Double-click on Windows.

Usage
1. Install dependencies: `npm install`
2. Edit `launcher/launcher-config.json` to add your `ngrokAuthtoken` and confirm `preferredTunnel`.
3. On macOS: `chmod +x mac/run-localserver.command` then double-click the file in Finder; a Terminal window will open and run the server.
4. On Windows: double-click `windows\run-localserver.bat` (you may pin it to the Start menu or create a shortcut).

Notes
- The launcher starts the server process (`node src/server/index.js`) in the project root and sets `TUNNEL` to the chosen tunnel.
- The server writes an ephemeral public URL to `LIVE_DEMO_URL.txt` and the launcher opens it automatically.
- Keep your `NGROK_AUTHTOKEN` private — prefer setting it in the `launcher-config.json` on your local machine or as an environment variable.

Packaging

- macOS (.app): use `npm run package:mac`. This runs `scripts/make-macos-app.sh` which creates `dist/RoomApp.app` — a minimal .app bundle that invokes the Node launcher. The resulting `.app` is a thin wrapper and requires Node to be installed on the target machine. To make it truly standalone, consider bundling Node or using a packaging tool.

- Windows (.exe): use `npm run package:win`. This runs `scripts/build-windows-exe.sh` which uses `pkg` (via `npx`) to build a single executable at `dist/RoomApp-win.exe`. `pkg` may not support every Node version or platform; run this on Windows or in an environment that supports cross-compilation for the target.

Notes

- Packaging commands may require additional developer tools on your machine (e.g., `pkg`, `npx`, or macOS toolchain). Test builds on the target OS.
- The macOS `.app` generated by `make-macos-app.sh` is intentionally minimal so you can inspect and modify it for signing or icons.
